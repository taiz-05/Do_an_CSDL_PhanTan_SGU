Vấn đề 1

Phân quyền Truy cập (Authorization)

Hệ thống tạo ra các User độc lập tại tầng Database. Ngay cả khi bị lộ mã nguồn hay bị tấn công SQL Injection ở ứng dụng Hành chính, Hacker cũng không thể lấy được Lương.

Tài khoản Nhân viên (Chỉ DB1)

-- Chỉ cấp quyền SELECT trên bảng công khai
CREATE USER 'nhanvien'@'%' IDENTIFIED BY '***';
GRANT SELECT ON db_hanhchinh.ThongTinChung 
TO 'nhanvien'@'%';

Truy vấn vào CSDL Lương sẽ lập tức bị báo lỗi Access Denied từ MySQL.

Tài khoản Kế toán (Chỉ DB2)

-- Chỉ cấp quyền quản lý trên DB Lương
CREATE USER 'ketoan'@'%' IDENTIFIED BY '***';
GRANT SELECT, UPDATE ON db_luong.LuongThuong 
TO 'ketoan'@'%';

Kế toán chỉ làm việc với các con số và Mã Nhân Viên, không thấy tên người nhận.

Vấn đề 2

Tính trong suốt (Transparency)

Mục tiêu
Quản trị viên (Admin) không cần biết dữ liệu đang bị phân mảnh ở đâu. Tại Trụ sở chính (DB1), Admin chỉ cần SELECT trên 1 bảng duy nhất để lấy ra toàn bộ thông tin (Tên + Lương).

Cấu hình FEDERATED Engine + VIEW

Tạo một bảng ảo ở DB1, bảng này âm thầm kết nối qua TCP/IP tới DB2. Sau đó dùng VIEW để JOIN dữ liệu vật lý và dữ liệu từ mạng.

-- 1. Tạo bảng ảo móc nối sang DB2
CREATE TABLE Remote_LuongThuong (
    MaNV INT PRIMARY KEY,
    LuongCoBan DECIMAL(15, 2),
    HeSo DECIMAL(4, 2)
) ENGINE=FEDERATED 
CONNECTION='mysql://link_user:***@mysql-luong:3306/db_luong/LuongThuong';

-- 2. Tạo View trong suốt cho Admin
CREATE VIEW vw_HoSoToanDien AS
SELECT a.MaNV, a.HoTen, a.PhongBan, b.LuongCoBan, b.HeSo
FROM ThongTinChung a
LEFT JOIN Remote_LuongThuong b ON a.MaNV = b.MaNV;
Vấn đề 3

Đồng bộ & Tính toàn vẹn (Transactions)

Khi thêm một nhân viên mới, dữ liệu phải được tự động "chẻ đôi" vào DB1 và DB2. Nếu DB2 bị sập đột ngột (mất mạng), hệ thống phải hủy bỏ (Rollback) thao tác trên DB1 để tránh rác dữ liệu.

Stored Procedure sp_ThemNhanVien

CREATE PROCEDURE sp_ThemNhanVien(...)
BEGIN
    -- Bắt lỗi: Nếu 1 trong 2 lệnh INSERT thất bại -> ROLLBACK tất cả
    DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
        ROLLBACK;
        RESIGNAL;
    END;

    START TRANSACTION;
        -- Insert Local (Thông tin hành chính)
        INSERT INTO ThongTinChung (MaNV, HoTen, PhongBan) VALUES (...);
        
        -- Insert Remote (Xuyên qua bảng Federated để vào DB2)
        INSERT INTO Remote_LuongThuong (MaNV, LuongCoBan, HeSo) VALUES (...);
    COMMIT;
END;

Bước 1: Giả lập Sập Server

Mở Docker Desktop, bấm STOP container mysql-luong. Điều này giả lập việc DB2 bị mất điện hoặc đứt cáp quang.

$ docker stop mysql-luong
mysql-luong exited with code 137

Bước 2: Ghi dữ liệu & Hủy bỏ

Trên Web, nhập Form thêm Mã 100 - Trần Lỗi Mạng và Lưu. Hệ thống báo lỗi. Kiểm tra lại danh sách Nhân viên (DB1):

  • Lệnh Insert DB1 ban đầu thành công.
  • Lệnh Insert DB2 (qua Federated) thất bại.
  • → Kích hoạt ROLLBACK xóa luôn bản ghi ở DB1. Không có rác dữ liệu!
